2) session
- httpsession 클래스의 객체
- 서버 메모리에 저장되는 정보
- 유효시간을 지님
- 주로 로그인 정보, 장바구니 등에 세션이 사용



& 스코프 종류
1. ServletContext : 동일한 애플리케이션에서 접근 가능(서버가 유지되는한 지속저장)
2. HttpSession   : 동일한 브라우저에서 접근 가능(웹 사이트 안에서 유지)
3. HttpServeletRequest : 하나의 요청/응답 사이클에서만 접근 가능(요청응답이 끝나면 사라짐)

& servlet attribute
- 서블릿 속성
- 웹 프로그램 실행 시 데이터를 서블릿 관련 객체에 저장
1) setattribute(string name object obj): 지정한 name 속성으로 obj저장
2) getattribute(string name)		: 지정한 name 속성의 값을 반환
3) removeattribute(String name)	: 지정한 name 속성 삭제

[jsp]
1. 정의 : 웹 서버에서 객체 또는 변수가 생성된 후 유효할 수 있는 범위를 말한다

2. 내장객체 종류(scope)
& 내장객체들은 별도로 선언하지 않아도 참조하여 사용가능하다
1) page: 페이지 영역에 저장한 값은 페이지를 벗어나면 사라짐.

2) request: redirect 시 사라짐. forward는 값유지
=> getparameter 메서드 사용가능

3) session: 인증된 사람만 허락 ,, ex) 로그인 유지
=> getwriter 메서드 사용가능

4) application: 전체 영역

5) pageContext : jsp 페이지 하나에 1:1로 연결된 객체
(하나의 jsp 페이지에 하나의 pageContext객체가 생성)
- 같은 페이지 내에서는 서로 값을 공유할 수 있으며,

- 정의 : jsp페이지의 컨텍스트 텍스를 의미한다

3. 공통함수
- scope 객체에 key, value 형식으로 값을 할당한다
1) setAttribute(String key, object value)
2) getAttribute(key)
