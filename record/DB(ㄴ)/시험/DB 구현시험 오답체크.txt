& 데이터베이스 구현 시험 오답체크

1. CMD에서 SQLPULS 접속 오류
=> 직접 오라클파일 경로를 입력 후 실행시키면 됨
ex) C:\Users\GDJ54>C:\oraclexe\app\oracle\product\11.2.0\server\BIN\SQLPLUS


2. 테이블 스페이스 생성
쿼리문 : CREATE TABLESPACE MY_SPACE DATAFILE 'C:\oracle\my_space.dbf' SIZE 10M;
=> 테이블 생성, 
     테이블명 : MY_SPACE,				* 테이블 명은 소문자면 소문자로 작성해도됨
     경로 : 'C:\oracle\my_space.dbf'
     용량 : 10M;					* 용량은 10MB가 아닌 10M을 입력해야됨


3. 데이터사전
1) 오라클 데이터베이스에서 발생한 모든 변경사항을 기록하는 리두로그 파일 목록 조회를 위한 데이터 사전의 이름
=> V$LOGFILE;
2) 관리자가 생성한 테이블 스페이스 목록 조회 데이터 사전
=> DBA_TABLESPACES
3) 완료보고서를 작성하기 위해 현재 사용자(USER)가 생성한 오브젝트의 목록을 조회
=> USER_OBJECTS


4. 새로운 사용자 생성과 권한부여
1) 사용자 생성
CREATE USER SPRING IDENTIFIED BY 1111 DEFAULT TABLESPACE MY_SPACE;
=> 디폴트 테이블 스페이스도 설정 가능 *


2) 권한 설정
GRANT CONNECT, RESOURCE TO SPRING;


5. 테이블 모델링(BAKER)
=> 칼럼값 아래에는 각종 제약사항들이 있으며, 제약사항의 이름부여 유무, 이름 등을 확인할 수 있다 ***


6. 테이블 생성 주의
1) PK를 작성해도 테이블 생성문에는 꼭 NOT NULL 작성하기 
2) FK의 참조에 테이블(칼럼) 꼭 붙이기
3) ALTER을 사용하는 이유 : ALTER을 사용하면 제약사항에 이름 명명이 가능해짐 ***
