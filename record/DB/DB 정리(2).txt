[1] DDL 테이블
1. 계정 초기화
1) 계정삭제
DROP USER 계정명 CASCADE;
=> 어떤 권한이 있든 계정 삭제

2) 계정 생성
CREATE USER 계정명 IDENTIFIED BY 비밀번호;
=> 계정과 비밀번호 생성

3) 권한 부여
GRANT CONNECT, RESOURCE TO 계정명;
=> 접속권한, 테이블수정, 관리권한을 부여

* 이 과정으로 정리해야할 테이블들을 번거롭지 않게 한꺼번에 리셋할 수 있음

2. 테이블 데이터 타입
1) char		: 용량(글자길이)가 고정된 타입
2) varchar2	: 용량이 가변되는 문자타입(대체로 사용)
3) number(p, s)	: 최대길이가 p, 소수점 이하 자리수가 s인 숫자타입
		* number : 최대 38자리(22바이트), 0은 불가능
4) date		: 날짜타입

3. 테이블 조회
1) 데이터 사전의 종류
(1) DBA_		: DB 모든 객체에 대한 접근
(2) USER_ 	: 현재 사용자가 정의한 객체
(3) ALL_ 		: 현재 사용자가 접근할 수 있는 객체

* 활용) SELECT TABLE_NAME FROM USER_TABLES;
=> 이런식으로 쿼리문에 넣어서 사용
