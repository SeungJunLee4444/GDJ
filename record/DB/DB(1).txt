4. 실행, 접속, 생성, 권한, 삭제 쿼리문
(1) 실행
=> run sql, 또는 cmd에서 sqlplus 입력 후 실행
(2) 접속
{1} sys 계정
/as sysdba : 시스템 관리하는 사람(oper보다 더 윗등급 권한을 지님)
/as sysoper: 시스템 사용하는 사람
{2} system 계정
conn system
conn system/비밀번호

(3) 생성
create user user1 identified by 2222
=> user1 계정을 만들고 2222라 비밀번호를 생성

(4) 권한
grant connect, resource, to scott
=> scott 유저에게 connect, resource 권한을 줘라(접속, 자원사용권한)
(5) 취소
revoke connect, resource from scott
=> scott 유저의 connect, resource 권한을 취소
(6) 삭제
drop user user1; 
=> user1을 삭제해라( * 만약 user1이 권한을 지니고 있으면 삭제안됨)
drop user user1 cascade;
=> user1이 가진 권한이 있든말든 계정삭제해라

* 쿼리문은 기본적으로 대문자를 사용
* 아이디는 대소문자 안가리고, 비밀번호는 대소문자를 가린다
