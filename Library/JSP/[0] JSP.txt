[JSP]
------
1. 서블릿의 한계
- 서블릿으로는 복잡한 화면을 만들기 어려움
- 자바를 알아야만 만들 수 있음

2. JSP
- 서블릿은 자바 안에 HTML을 포함
=> JSP는 HTML기반, 안에 자바를 넣을 수 있게 만듬

& 서블릿과 jsp의 차이
1) 서블릿 : 자바의 영역인 패키지에 생성, JSP : 웹의 영역인 webapp에 생성
2) 서블릿 : html문서는 web문서 영역에, 서블릿은 자바 영역에
    jsp     : 웹문서 영역에 전부?

1) 동작
JSP실행 -> 서블릿으로 변환 -> 컴파일
- 서블릿을 돌리는것과 동일하지만, JSP를 서블릿으로 자동변환하는 과정이 추가된다
=> <div>가 -> out.println("<div>")로 자동변환

& 상단 DIRECTIVE에, <%@ page language="java" contentType="text/html; charset=UTF-8"
    pageEncoding="UTF-8"%>


2) JSP의 구성요소
- HTML, CSS, JS
+ JSP용 기본 태그, 액션태그, 커스텀 태그, 스크립트 요소
